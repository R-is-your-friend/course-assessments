Working with R Objects

Types of Objects in R

This week you’ve learned how to make four different types of objects in R:

  • vectors
  • matrices
  • lists
  • dataframes

Let’s review each of these so they are fresh in our memory!

Vectors

Suppose you want to create two vectors:

  1. The days of the week
  2. The temperature of the water at Pismo each day

Pencil out what code you would write to create these vectors.

Creating Vectors

days <- c("Monday", 
          "Tuesday", 
          "Wednesday", 
          "Thursday", 
          "Friday", 
          "Saturday", 
          "Sunday")
temps <- c(60.1, 
           59.4, 
           60.8, 
           61, 
           60.1, 
           62.1, 
           59.7)

. . .


What would you expect this code to output?

typeof(days)
typeof(temps)

Lists

Unlike matrices, lists allow the elements to have different data types and different sizes.


Here are two ways to make a list:

watertemp_list <- list(days, 
                       temps)
watertemp_list <- list(days = days, 
                       temps = temps)


How are these lists different?

Dataframes

A dataframe is a special type of list—where every element (column) is the same size.


Draw out how the watertemp_df would look:

watertemp_df <- data.frame(days = days, 
                           temps = temps)

Modifying an Object

Suppose we wanted to add a column containing information on the air temperature to watertemp_df.


Here are the air temperatures for each day: 67, 65, 67, 70, 69, 68, 68

watertemp_df$airtemp <- c(67, 65, 67, 70, 69, 68, 68)

Modifying an Object – Changing Specific Values

We also learned that we got the water temperature wrong on Thursday and Friday.

On Thursday the water temperature was actually 60.8 and on Friday the temperature was actually 61.2.


Pencil out how you would adjust these water temperature values.

Modifying an Object – Changing Specific Values

  • Thursday is row 4 and Friday is row 5
  • water temperature is column 2 (named temps)

Option 1:

watertemp_df[4:5, 2] <- c(60.8, 61.2)

Option 2:

watertemp_df[4:5, "temps"] <- c(60.8, 61.2)

Option 3:

watertemp_df$temps[4:5] <- c(60.8, 61.2)

Modifying an Object – Changing Column Values

We originally store the air and water temperatures in degrees Fahrenheit, but many people prefer degrees Celsius.

The formula for converting from Fahrenheit to Celsius is:

\[\text{Celsius} = \frac{5}{9}(\text{Fahrenheit} - 32)\]

Pencil out how you would convert the temps and airtemp columns to Celsius.

Modifying an Object – Changing Column Values

Option 1: Adjust the temperatures and save them back in the original column

watertemp_df$temps <- (5/9) * (watertemp_df$temps  - 32)
watertemp_df$airtemp <- (5/9) * (watertemp_df$airtemp  - 32)


Option 2: Make a new column with the Celsius temps

watertemp_df$temps_c <- (5/9) * (watertemp_df$temps  - 32)
watertemp_df$airtemp_c <- (5/9) * (watertemp_df$airtemp  - 32)

Assessing Values of an Object

Personally, I won’t swim at Pismo if the temperature is below 16 degrees (Celsius). How many days in the last week would I have swam?

Pencil out the code you would write to answer this question.

  1. What relational comparison will you use?
  2. What output will this comparison give you?
  3. How will you use this output to find the number of days I would have swam?

How many days would I have swam?

  1. What relational comparison will you use?
watertemp_df$temps >= 16
  1. What output will this comparison give you?
[1] FALSE FALSE  TRUE  TRUE  TRUE  TRUE FALSE
  1. How will you use this output to find the number of days I would have swam?
sum(watertemp_df$temps >= 16)

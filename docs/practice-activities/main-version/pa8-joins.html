Today we will be exploring “twin towns” (also called sister cities). These are cities that have some sort of social or political relationship with each other, usually for the purpose of cultural exchange. A list of twin towns is kept up on Wikipedia, which is the source of our data today!

The data come to us from TidyTuesday – originally compiled by Ahmad Barclay and further curated by Nicola Rennie for TidyTuesday.

We also created a dataset dists with the following columns:

  • source: same as in links
  • target: same as in links
  • dist: distance between paired cities in km
cities <- read_csv('https://raw.githubusercontent.com/rfordatascience/tidytuesday/main/data/2026/2026-05-12/cities.csv')

links <- read_csv('https://raw.githubusercontent.com/rfordatascience/tidytuesday/main/data/2026/2026-05-12/links.csv')

dists <- read_csv('https://raw.githubusercontent.com/r-is-your-friend/course-assessments/main/data/08-dists.csv')
  1. Check out the data. What is one observation in the cities dataset and what kind of information do we get from this data? What is one observation in the links dataset and what kind of information do we get from this data?

The links data contains all of the pairs of cities, but we can’t learn anything interesting just from the city IDs! We want to add the city information on.

  1. What are the key(s) between the city and links data?

As the data dictionary states, the order of “source” and “target” city is arbitrary.

  1. Created a new dataframe called linked_cities that includes all of the information in the cities data for both cities in each pair of twinned cities.

Note: This means you will have two name, latitude, longitude, etc. columns for each row. Think about how you might want to name columns to keep things from being confusing!

# Q3 code
linked_cities <- links |> 
  left_join(cities, join_by(source == id)) |> 
  rename_with(.fn = \(x) paste0(x, "_1"), 
              .cols = name:continent) |> 
  left_join(cities, join_by(target == id)) |> 
  rename_with(.fn = \(x) paste0(x, "_2"), 
              .cols = name:continent)
  1. Are any of the twinned cities missing city information like their name?
# Q4 code
linked_cities |>

  1. Q4 implies that we could have used which of the following joins (or all of them)? inner, left, right, and/or full?

We didn’t find that any of the twinned cities were missing information from the cities data, keeping all of the linked_cities observations, so we definitely could have used inner or left join with the same outcome, (assuming linked_cities was the left dataset). If you did some more investigating, you would find that the cities data does not include any cities that are not in the linked_cities data as well. Since the same exact keys are present in both datasets, any of the joins would have resulted in the same outcome here!

  1. Join the distances between cities on to your linked_cities data. Pay close attention: what identifies one observation in the dists data? Your linked_cities data should still have 10,596 observations.
# Q6 code
linked_cities <- linked_cities |> 
  left_join(dists, 
            by = join_by(source, target))
